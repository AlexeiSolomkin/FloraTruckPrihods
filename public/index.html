📦 Сервис для автозаполнения Приходов

Загрузи Табель и шаблон Прихода — получи заполненный файл.

Как это работает

Что умеет сервис (кратко, подробности - в блоках ниже):

• Считывать нужную информацию из Табеля и распределять груз по клиентам в Приход.

• Автоматически заполнять из Табеля номер машины в Приход, а также проверять совпадение номера машины в Табеле и в названии Прихода.

• Учитывать клиентов с исключениями (у кого выставляется нетто вес и не ставится сертификат).

• Корректно обрабатывать грузы, разбитые на несколько машин.

• Корректно обрабатывать одноклиентские консолидации.

• Корректно выставлять авб, у которых первая из последних 4 цифр - это ноль(выставляется как ".0123").

Что НЕ умеет сервис:

• Работать с голландским Табелем. (в разработке)

• Работать с клиентами БЕЗ сертификата (не путать с теми клиентами, у кого сертификат не выставляется) - пока что нужно вручную заполнять табель для клиентов без сертификата, а также удалять их из Прихода с сертификатом, т.к. алгоритм считает их за клиентов с сертификатом.

• Выставлять прикулинг.

• Корректно обрабатывать консолидации, отличные от BOG и UIO.

• Исправлять ошибки, допущенные при составлении Табеля :(

Что нужно загрузить:

Шаг 1 — Табель: файл вида «Табель №АМ и для склада.xlsx». Программа читает из него нужную информацию для заполнения Прихода.

Шаг 2 — Пустой Приход: шаблон «01.01.2026 Приход ... ам №АМ СЕРТ ... .xlsx». Данные вписываются в листы каждого клиента — стили, цвета и формулы остаются нетронутыми.

Правила распределения груза (консолидации, миксы, одиночные клиенты)по таблицам:

• Консолидация + страна UIO → блок КОНСОЛЬ ЭКВАДОР (строки 35–37)

• Консолидация + страна BOG → блок КОНСОЛЬ КОЛУМБИЯ (строки 42–45)

• Миксы и одиночные клиенты → блок ИМПОРТ АМС-МСК (строки 50–52)

• Сертификат (B29) = количество АВБ в блоке ИМПОРТ. Исключения — клиенты без серта — об этом ниже.

Номер машины заполняется автоматически

Берётся из ячейки «Номер машины» внутри Табеля. Записывается в ячейку C2 листа «!!!!!» Прихода — все клиентские листы подтягивают его автоматически через формулу.

Если номера из названия Табеля и Прихода не совпадают — программа покажет предупреждение и спросит, продолжить ли составление прихода.

Исключения по весу (нетто)

Клиенты из списка исключений получают нетто вес вместо брутто — но только для блока ИМПОРТ. Консолидации всегда брутто. Если нетто не заполнено в Табеле — берётся брутто с предупреждением в консоли.

Груз разбит на несколько машин

Если груз едет в двух машинах, в Табеле над суб-таблицей(суб-таблицы - это таблицы, которые идут ниже основной в Табеле) ставится жёлтая строка «В ПРИХОДЕ АМ №АМ». Программа читает этот маркер:

• Если номер совпадает с номером текущей машины → суб-таблица включается в Приход как обычно.

• Если номер не совпадает с номером текущей машины → вся суб-таблица пропускается, в Приход не вносится.

Обе ситуации выводятся в консоль в разделе «Груз разбит на машины».

Ограничения

• Строка вида «консолидация Имя клиента» с непустым номером клиента обрабатывается автоматически: попадает в блок КОНСОЛЬ ЭКВАДОР или КОНСОЛЬ КОЛУМБИЯ.

• Жёлтые строки без номера клиента и без суб-таблицы пропускаются.

• Если у клиента больше 10 АВБ в одном блоке — программа предупредит: проверьте диапазон формулы TOTAL в шаблоне.

• Списки исключений (нетто / без учета серта) действуют только в текущей сессии браузера. При перезагрузке страницы вернутся стандартные(Функция полноценного добавления будет добавлена позднее).
